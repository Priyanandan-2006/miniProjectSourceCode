# Smart Transaction Processing System

This project upgrades the original textbook-style `trans.c` into a more demo-ready banking console app with better reliability, safer input handling, and stronger presentation value for a mini-project or hackathon demo.

## Changes Made in `trans.c`

-You turned a basic account program into a full banking system with 12 features (like search, transfer, reports, etc.).
-The program now auto-creates and fixes the data file (credit.dat) if it’s missing or corrupted.
-You organized data handling using proper functions, so reading/writing accounts is cleaner and safer.
-You replaced unsafe inputs with validated input methods to avoid crashes and wrong data.
-You added rules to prevent mistakes like:
   duplicate accounts
   invalid account numbers
   negative balance
   overdraft
   transferring to the same account
-The account list now shows which customers have low balance.
-You can search accounts easily by number or name (case-insensitive).
-You added money transfer between accounts.
-You included bank statistics, like:
    total money in bank
    average balance
    highest balance account
    number of active accounts
-You added a top customers ranking (leaderboard).
-You created a customer insight system (basically smart analysis + suggestions for each user).
-Every action is now logged in transactions.log with time.
-You can view recent transactions inside the program itself.
-You improved export so only valid accounts are saved neatly in accounts.txt.
## What Was Improved

- Fixed unsafe menu and numeric input handling by replacing raw `scanf`-driven flows with validated line-based parsing.
- Repaired the broken `accounts.txt` export logic that previously produced garbage rows for malformed binary data.
- Added automatic data-file normalization so `credit.dat` is always kept at 100 valid account slots.
- Prevented invalid operations such as overdraft withdrawals, duplicate account creation, and self-transfers.

## New Features

- List all active accounts with a balance health status.
- Search accounts by account number, first name, or last name.
- Transfer funds between two accounts.
- Show an analytics dashboard:
  - total active accounts
  - total funds in the system
  - average balance
  - low-balance account count
  - top account by balance
- Show a top-customer leaderboard ranked by balance.
- Generate a customer insight card with:
  - customer segment
  - financial wellness score
  - 1-year balance projection
  - loan-readiness suggestion
  - smart savings alert
- View the most recent transactions from inside the app.
- Append every create, update, delete, and transfer action to `transactions.log`.

## Files

- `trans.c` - main source code
- `credit.dat` - binary random-access account storage
- `accounts.txt` - formatted export/report generated by the program
- `transactions.log` - audit trail of account activity

## Build

```bash
gcc -Wall -Wextra -std=c11 trans.c -o trans_hackathon.exe
```

## Run

```bash
./trans_hackathon.exe
```

## Demo Flow Suggestion

1. List active accounts.
2. Add a new account.
3. Transfer funds between two accounts.
4. Search for the new customer.
5. Show the analytics dashboard.
6. Show the leaderboard and customer insight card.
7. Open recent transactions.
8. Export `accounts.txt` and open the report.
